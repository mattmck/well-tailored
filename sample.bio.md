# Bio

I'm a software engineer who likes building practical software that solves real problems, especially tools that make other people's work easier. Most of my career has been in backend and platform engineering — distributed systems, data pipelines, developer tooling, and the plumbing that keeps production reliable. I do my best work when I can own something end to end, from design through delivery, and I care about leaving systems in better shape than I found them.

I'm looking for roles where I can stay close to both the code and the product. I'm drawn to developer experience, internal platforms, backend services, and product engineering where there's a clear user or business outcome. I usually do well in startup and mid-sized environments where engineers have real ownership and can move quickly. I'm open to remote-friendly or hybrid teams with strong written communication and a thoughtful engineering culture.

My work style is collaborative and pragmatic. I care about shipping, but also about the fundamentals that make software sustainable: clean code, sensible abstractions, good documentation, reliable release processes, and eliminating unnecessary complexity.

Outside of my main work, I've been building AI-integrated tools as personal projects. I've worked with several LLM APIs and built batch and streaming workflows optimized for cost and latency. My personal projects focus on semantic search and classification over domain-specific corpora, running on serverless cloud infrastructure.
